# C2C 交易所 API 接口文档

## 1. 概述

本接口用于业务系统与 C2C 交易所服务端进行数据交互，包含用户身份验证及资产划转功能。

- **基础 URL**: https://m.biyiex.top/api/c2c` (开发环境)
- **传输格式**: `application/JSON` 
- **安全认证**: 所有请求需包含签名 `sign` 参数。

------

## 2. 安全与签名机制

### 2.1 签名生成规则

1. 将所有请求参数（除 `sign` 外）按照参数名 ASCII 码从小到大排序。
2. 使用 `Key=Value` 的格式拼接成字符串，多个参数间用 `&` 连接。
3. 在拼接后的字符串最后加上商户密钥 `SECRET_KEY=W3Z6diRzXmcmSmpKEEpDREpLS0pLSPpLR0ZEU0ExMjM0NTY3ODk=`。
4. 对最终字符串进行 MD5 或 HmacSHA256 加密（具体取决于 `C2CSignUtils` 实现）并转换为大写。
5. java实现参考

    public static String sign(Map<String, Object> params) {

        String SECRET_KEY = SyConfigUtil.getConfigVal("SECRET_KEY");
        SECRET_KEY = StrUtil.isEmpty(SECRET_KEY) ? "W3Z6diRzXmcmSmpKEEpDREpLS0pLSPpLR0ZEU0ExMjM0NTY3ODk" : SECRET_KEY;
        // 1. ASCII排序
        TreeMap<String, Object> sortedMap = new TreeMap<>(params);

        // 2. key=value拼接
        String paramStr = sortedMap.entrySet()
                .stream()
                .map(e -> e.getKey() + "=" + e.getValue())
                .collect(Collectors.joining("&"));

        // 3. 拼接SECRET_KEY
        paramStr = paramStr + "&SECRET_KEY=" + SECRET_KEY;
        // 4. MD5加密并转大写
        return DigestUtil.md5Hex(paramStr).toUpperCase();
    }

## 3. 接口详情

### 3.1 用户身份验证 (自动登录)

通过业务系统的用户 ID 和 Token 在交易所侧进行身份同步与验证。

- **接口地址**: `/user/auto_login`
- **请求方式**: `POST`
- **功能说明**: 验证用户合法性并返回交易所侧的用户信息。

#### 请求参数

| **参数名** | **类型** | **必选** | **描述**                      |
| ---------- | -------- | -------- | ----------------------------- |
| bourseId   | String   | 是       | 业务系统用户唯一标识 (UserId) |
| type       | String   | 是       | 固定值：`LOGIN`               |
| token      | String   | 是       | 用户登录凭证                  |
| timeStamp  | String   | 是       | 当前毫秒时间戳                |
| sign       | String   | 是       | 签名字符串                    |

#### 响应参数说明 (Response Parameters)

##### 基础结构

所有接口均遵循统一的响应包装格式：

| **参数名** | **类型** | **描述**                            |
| ---------- | -------- | ----------------------------------- |
| **code**   | Integer  | 状态码（200表示成功，其他表示失败） |
| **msg**    | String   | 提示消息（操作成功或错误描述）      |
| **data**   | Object   | 业务核心数据对象                    |

------

#####  Data 对象详情 (用户信息)

`data` 字段内包含的具体参数如下：

| **参数名**     | **类型** | **示例值** | **描述**                                                |
| -------------- | -------- | ---------- | ------------------------------------------------------- |
| **bourseId**   | String   | `"1001"`   | **交易所用户ID**：对应系统中的唯一身份标识              |
| **bourseName** | String   | `"张三"`   | **用户名称**：用户在交易所登记的姓名/昵称               |
| **realName**   | Integer  | `1`        | **实名状态**： • `0`: 未实名 • `1`: 已实名              |
| **status**     | Integer  | `1`        | **用户状态**： • `0`: 冻结（禁止交易/访问） • `1`: 正常 |

------

#####  响应示例 (JSON)

```
{
    "code": 200,
    "msg": "操作成功",
    "data": {
        "bourseId": "1001",
        "bourseName": "张三",
        "realName": 1,
        "status": 1
    }
}
```





### 3.2 资产划转

用于在交易所内部进行资产转移（如从用户余额划转至订单锁定区），支持幂等校验。

- **接口地址**: `/wallet/transfer_asset`
- **请求方式**: `POST`
- **功能说明**: 保证扣除和增加的原子性，支持外部订单号幂等。

#### 请求参数

| **参数名** | **类型** | **必选** | **描述**                                |
| ---------- | -------- | -------- | --------------------------------------- |
| orderNo    | String   | 是       | 外部业务订单号 (和type联合用于幂等)     |
| bourseId   | String   | 是       | 出账方用户 ID                           |
| coinSymbol | String   | 是       | 币种符号 (如: USDT)                     |
| amount     | String   | 是       | 划转数量 (需为 String 类型以防精度丢失) |
| remark     | String   | 否       | 业务备注                                |
| type       | String   | 是       | 划转类型/业务方向                       |
| nonceStr   | String   | 是       | 随机字符串 (UUID)                       |
| timeStamp  | String   | 是       | 当前毫秒时间戳                          |
| sign       | String   | 是       | 签名                                    |

#### type类型说明

| 标识符 (Constant)           | 编码 (Code)               | 描述 (Description) | 业务场景 / 动作流程                                          |
| --------------------------- | ------------------------- | ------------------ | ------------------------------------------------------------ |
| **MERCHANT_DEPOSIT_DEDUCT** | `MERCHANT_DEPOSIT`        | 缴纳保证金         | 商家入驻时，从交易所可用余额扣除并进入保证金锁定池。         |
| **MERCHANT_DEPOSIT_RETURN** | `MERCHANT_DEPOSIT_RETURN` | 退回保证金         | 商家退出时，将锁定池中的金额释放回交易所可用余额。           |
| **ADV_PUBLISH_FREEZE**      | `ADV_PUBLISH_FREEZE`      | 发布广告资产冻结   | 商家挂“卖出”广告时，预先锁定其名下对应的币种资产。           |
| **ADV_CANCEL_UNFREEZE**     | `ADV_CANCEL_UNFREEZE`     | 下架广告资产释放   | 广告下架或结束时，将剩余未成交的冻结资产返还商家。           |
| **USER_SELL_FREEZE**        | `USER_SELL_FREEZE`        | 用户卖币下单冻结   | 普通用户向商家卖币时，下单瞬间冻结用户的币，防止“一币多卖”。 |
| **ORDER_RELEASE_TRANSFER**  | `ORDER_RELEASE_TRANSFER`  | 订单完成资产划转   | **核心动作**：卖家点击“放行”，资产正式从卖家冻结区划转至买家可用区。 |
| **ORDER_CANCEL_UNFREEZE**   | `ORDER_CANCEL_UNFREEZE`   | 订单取消资产释放   | 订单超时或主动取消后，将冻结的订单资产退还给卖家可用余额。   |

#### 响应码说明

| **Code** | **说明** | **处理建议**                     |
| -------- | -------- | -------------------------------- |
| 200      | 划转成功 | 流程继续，返回 data 为交易流水号 |
| 402      | 余额不足 | 提示用户充值或检查账户状态       |
| 403      | 资产异常 | 账号被冻结，需人工介入           |
| 409      | 重复请求 | 幂等命中，视为成功，无需重复操作 |
| 500      | 系统错误 | 交易所内部故障，建议稍后重试     |

#### 返回结果

JSON

```json
{
    "code": 200,
    "msg": "操作成功",
    "data": "TX1234567890" 
}
```
