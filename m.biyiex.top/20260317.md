# C2C 交易所 API 接口文档

## 1. 概述

本接口用于业务系统与 C2C 交易所服务端进行数据交互，包含用户身份验证及资产划转功能。

- **基础 URL**: https://m.biyiex.top/api/c2c` (开发环境)
- **传输格式**: `application/JSON` 
- **安全认证**: 所有请求需包含签名 `sign` 参数。

------

## 2. 安全与签名机制

### 2.1 签名生成规则

1. 将所有请求参数（除 `sign` 外）按照参数名 ASCII 码从小到大排序。
2. 使用 `Key=Value` 的格式拼接成字符串，多个参数间用 `&` 连接。
3. 在拼接后的字符串最后加上商户密钥 `SECRET_KEY=W3Z6diRzXmcmSmpKEEpDREpLS0pLSPpLR0ZEU0ExMjM0NTY3ODk`。
4. 对最终字符串进行 MD5 或 HmacSHA256 加密（具体取决于 `C2CSignUtils` 实现）并转换为大写。
5. java实现参考

    public static String sign(Map<String, Object> params) {

        String SECRET_KEY = SyConfigUtil.getConfigVal("SECRET_KEY");
        SECRET_KEY = StrUtil.isEmpty(SECRET_KEY) ? "W3Z6diRzXmcmSmpKEEpDREpLS0pLSPpLR0ZEU0ExMjM0NTY3ODk" : SECRET_KEY;
        // 1. ASCII排序
        TreeMap<String, Object> sortedMap = new TreeMap<>(params);

        // 2. key=value拼接
        String paramStr = sortedMap.entrySet()
                .stream()
                .map(e -> e.getKey() + "=" + e.getValue())
                .collect(Collectors.joining("&"));

        // 3. 拼接SECRET_KEY
        paramStr = paramStr + "&SECRET_KEY=" + SECRET_KEY;
        // 4. MD5加密并转大写
        return DigestUtil.md5Hex(paramStr).toUpperCase();
    }

# 接口文档：查询用户资产余额

## 1. 接口描述

**功能描述**：根据用户 ID 和币种标识，从资产系统实时获取该用户的可用余额。

**请求方式**：`POST`

**接口路径**：`/user/getBalance`（基于 `BASE_URL`）

------

## 2. 请求参数 (Parameters)

### 请求体 (Body) - `applicationJSON`

| **参数名称**   | **类型** | **必选** | **说明**                 | **示例**        |
| -------------- | -------- | -------- | ------------------------ | --------------- |
| **bourseId**   | Long     | 是       | 用户 ID / 账户 ID        | `10086`         |
| **coinSymbol** | String   | 是       | 币种标识（大写）         | `USDT`          |
| **nonceStr**   | String   | 是       | 随机字符串（32位 UUID）  | `5f2e...8d3a`   |
| **timeStamp**  | String   | 是       | 当前毫秒级时间戳         | `1710594234000` |
| **sign**       | String   | 是       | 数据签名（详见签名算法） | `A1B2C3...`     |

------

## 4. 响应参数 (Response)

| **字段名称** | **类型** | **说明**                             | **示例**                     |
| ------------ | -------- | ------------------------------------ | ---------------------------- |
| **code**     | Integer  | 状态码（200 为成功，其他为失败）     | `200`                        |
| **msg**      | String   | 错误描述信息                         | `查询余额成功`               |
| **data**     | String   | 用户余额（字符串形式，避免精度丢失） | `1250.45000000` /没有就返回0 |

------

## 5. 错误码说明 (Error Codes)

| **错误码** | **描述**     | **排查建议**                     |
| ---------- | ------------ | -------------------------------- |
| **200**    | 成功         | -                                |
| **500**    | 系统繁忙     | 资产系统内部异常，查看资产端日志 |
| **401**    | 签名验证失败 | 检查 `SECRET_KEY` 及签名排序逻辑 |

------

## 6. 接口示例

### 请求示例 (JSON)

JSON

```json
{
    "bourseId": 10086,
    "coinSymbol": "USDT",
    "nonceStr": "7b0d238c913444989679198642349021",
    "timeStamp": "1710594234000",
    "sign": "E10ADC3949BA59ABBE56E057F20F883E"
}
```

### 成功响应示例

```json
{
    "code": 200,
    "msg": "success",
    "data": "5000.1234"
}
```

### 失败响应示例

```json
{
    "code": 500,
    "msg": "用户账户已冻结",
    "data": null
}
```

# 接口文档：C2C 资产划转 (V2)

## 1. 接口描述

**功能描述**：在资产系统中执行两个用户之间的资产划转。支持幂等性校验，防止网络抖动导致的重复扣款。常用于 C2C 放行确认。

**请求方式**：`POST`

**接口路径**：`/asset/transfer2`（基于 `BASE_URL`）

------

## 2. 请求参数 (Parameters)

### 请求体 (Body) -json

| **参数名称**   | **类型** | **必选** | **说明**                               | **示例**                     |
| -------------- | -------- | -------- | -------------------------------------- | ---------------------------- |
| **orderNo**    | String   | 是       | 业务订单号（资产系统用于**幂等判断**） | `ORD20240316001`             |
| **fromUserId** | Long     | 是       | 出账方用户 ID（卖家）                  | `8001`                       |
| **toUserId**   | Long     | 是       | 入账方用户 ID（买家）                  | `9001`                       |
| **coinSymbol** | String   | 是       | 币种标识                               | `USDT`                       |
| **amount**     | String   | 是       | 划转数量（建议使用 `toPlainString()`） | `150.00`                     |
| **type**       | Integer  | 是       | 业务类型（放行、扣减、纠纷处理等）     | `**ORDER_RELEASE_TRANSFER**` |
| **remark**     | String   | 否       | 业务备注                               | `transferAsset`              |
| **nonceStr**   | String   | 是       | 随机字符串                             | `a1b2c3d4...`                |
| **timeStamp**  | String   | 是       | 毫秒级时间戳                           | `1710594234000`              |
| **sign**       | String   | 是       | 签名结果                               | `F3E2...`                    |

------

## 3. 响应参数 (Response)

| **字段名称** | **类型** | **说明**                              | **示例**   |
| ------------ | -------- | ------------------------------------- | ---------- |
| **code**     | Integer  | 状态码（200 为成功）                  | `200`      |
| **msg**      | String   | 错误或提示信息                        | `操作成功` |
| **data**     | String   | **资产系统流水号 (txId)** 或 幂等标识 | `TX990283` |

------

## 4. 状态码 (Code) 业务含义

| **状态码** | **业务含义** | **处理逻辑建议**                                      |
| ---------- | ------------ | ----------------------------------------------------- |
| **200**    | **划转成功** | 流程继续，更新本地订单为“已完成”。                    |
| **402**    | **余额不足** | 停止操作，提示用户账户余额或冻结额度不足。            |
| **403**    | **资产异常** | 停止操作，提示用户联系客服（可能账户被封禁）。        |
| **409**    | **重复请求** | **幂等命中**。说明该 orderNo 已处理过，视为成功处理。 |
| **500**    | **系统错误** | 资产系统内部异常，建议人工介入或稍后重试。            |

------
