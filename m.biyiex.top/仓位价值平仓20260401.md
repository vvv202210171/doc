# 仓位价值平仓 接口文档（2026-01-01）

## 接口摘要
- 接口路径：`POST /v2/contract/close`
- 功能：按输入的仓位价值（USDT）部分或全部平仓
- 认证：需要登录（从请求中获取当前会员号）

## 请求头
- `Content-Type: application/json`
- `token: biyixxxx`  （或使用项目中通用的登录凭证方式）

## 请求体（JSON）
```json
{
  "autoid": "string",    // 必填，订单 id 
  "usdtNum": "string"    // 必填，要平仓的仓位价值（字符串形式的十进制数，单位：USDT）
}
```

## 参数校验规则
1. `autoid` 与 `usdtNum` 均为必填，不能为空。
2. `usdtNum` 必须为大于 0 的数值（服务器内部以 `BigDecimal` 解析并判断）。
3. 若校验不通过，返回错误（项目中抛出 `ExWebException(1, LanguagesUtil.get("parameter"))`）。

## 业务逻辑（简要）
1. 根据当前登录用户和 `autoid` 查询订单；若订单不存在或不属于当前用户，抛出错误（`LanguagesUtil.get("noOrder")`）。
2. 将请求的 `usdtNum` 转为 `BigDecimal`：
   - 若 `usdtNum` >= `order.getPosition()`，则视为全仓平仓：调用 `contract.close_position(autoid, member, order.getUnfilled())` 并返回订单信息。
3. 否则根据交易对信息计算要平仓的数量（张数）：
   - 从交易对配置取 `unit` 与 `amountAccuracy`，计算方法为：
     number = getNumber(usdtNumBig, order.getPrice(), contractCoinPo.getUnit(), contractCoinPo.getAmountAccuracy())
   - 如果计算得到的张数 >= `order.getUnfilled()`，也视为全仓平仓（调用 `close_position` 并返回订单）。
   - 否则按计算的张数调用 `close_position(autoid, member, number)` 并返回订单。

## 返回值（成功）
成功返回项目统一包装 `R<ContractOrderPo>`，其中 `data` 为被操作的订单对象（`ContractOrderPo`）。

示例：
```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "autoid": "order123",
    "member": "member001",
    "maincoin": "USDT",
    "tradcoin": "BTC",
    "position": "1000.00",
    "unfilled": "2.00",
    "price": "50000.00"
    // 其它订单字段 ...
  }
}
```

## 错误示例（常见）
- 参数错误（缺失或 `usdtNum` 非正数）：
```json
{
  "code": 1,
  "msg": "parameter"
}
```
- 订单不存在或不属于当前用户：
```json
{
  "code": 1,
  "msg": "noOrder"
}
```
- 其它由底层服务抛出的错误（例如交易对信息异常、单位为 0 等），会返回相应的错误码与消息（参照项目的 `ExWebException` 处理与 `LanguagesUtil` 文案）。

## 注意事项
1. `usdtNum` 应以字符串形式传递以保证精度，服务器内部以 `BigDecimal` 处理。
2. 计算张数时使用 `getNumber` 中的舍入策略（本代码中为向上取整，RoundingMode.UP），以避免向下取整导致用户实际金额不足。
3. 若 `usdtNum` 足够大会触发全仓平仓（按未平仓数 `order.getUnfilled()` 执行平仓）。
4. 平仓与账务变更需要原子事务保证，`close_position` 内部应处理事务与日志，异常将回滚。
