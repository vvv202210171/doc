# 合约USDT结算（前端使用文档）

说明：本页面描述后端控制器 `ContractV2Controller`（路径：`/v2/contract`）提供的合约下单接口（USDT 结算方式）。文档以前端集成为目的，包含请求格式、参数说明、服务器端计算规则、校验与常见错误。文档名称：合约USDT结算。

---

## 概览
- 基础路由：`/v2/contract`
- 支持的接口（均为 POST）：
  - `/buyLimit`  — 限价买入（USDT 结算）
  - `/sellLimit` — 限价卖出（USDT 结算）
  - `/buyMarket` — 市价买入（USDT 结算）
  - `/sellMarket` — 市价卖出（USDT 结算）

- 认证：接口通过后端从请求中获取用户信息（`HelpOpenUser.getMemberNo(request)`）。前端需保证登录态 / token（cookie/header）能被后端识别。
- 返回格式：统一使用 `R`（后端返回包装），成功示例通常为：
  ```json
  { "code": 0, "data": { ... }, "msg": "" }
  ```
  失败示例（抛出 `ExWebException`）：
  ```json
  { "code": 1, "msg": "错误原因文本" }
  ```

## 公共请求参数说明（适用于所有四个接口）
- maincoin (string) — 主币种，例如 `USDT`（必填）
- tradcoin (string) — 交易币种，例如 `BTC`（必填）
- usdtNum (string) — 用户投入的 USDT 总仓位价值（字符串形式，必填，必须 > 0）
- lever (string) — 杠杆倍数（字符串形式，必填）
- tdMode (string) — 交易模式（逐仓/全仓），传值请与后端约定（前端使用与后端一致的枚举值）

额外（限价接口需要）：
- price (string) — 限价委托的下单价格（字符串形式，必填，必须 > 0）

可选：
- profitPrice (string) — 止盈价（可空，不填则不设置）
- lossPrice (string) — 止损价（可空，不填则不设置）

注意：控制器内部会根据 `maincoin` 与 `tradcoin` 从合约配置读取每手合约的 `unit`、数量精度（amountAccuracy）、price 精度等。

---

## 计算规则（非常重要，前端需知）
后端将以 `usdtNum` 与 `price`（限价）或合约当前行情价（市价）计算实际买入的合约手数（张数），计算公式如下：

- 单张价值 = price * unit
- 实际买入手数（number） = floor( usdtNum / (price * unit) , amountAccuracy)
  - floor 意味着向下取整（RoundingMode.DOWN）到 `amountAccuracy` 小数位
  - 后端会检查并保证：number >= min && number <= max（合约配置中的最小/最大手数）

示例（假设）：
- price = 20,000
- unit = 0.001
- usdtNum = 100
- 单张价值 = 20000 * 0.001 = 20 USDT
- number = floor(100 / 20) = 5 张

实现要点：
- 如果 price×unit == 0 或 contract 配置异常，后端会返回错误（避免除以零）
- 舍入采用向下取整，保证不会因为四舍五入导致用户实际所需 USDT 超出输入的 usdtNum

## 校验规则与常见错误（后端 key）
下列为后端在校验失败时常用的错误 key（后端会通过 `LanguagesUtil.get(key)` 返回本地化文本），前端可据此显示提示或做前端校验以减少回退：

- `parameter` — 参数缺失或格式错误（必填项为空）
- `errorbydata` — 输入值数据错误（如非数字或 <= 0）
- `noCoin` — 交易对不存在或未上线
- `error` — 一般错误（如合约配置异常、unit 为 0 等）
- `minusdt` — 计算得到的手数小于允许最小手数（返回的 message 通常含具体最小金额）
- `maxusdt` — 计算得到的手数大于允许最大手数（返回的 message 通常含具体最大金额）
- `noEffective` — 用户不是有效合约交易账户（无权限下单）
- `shortLower` — 卖空（SELL）方向价格低于当前价的校验错误
- `longHigher` — 买多（BUY）方向价格高于当前价的校验错误
- `limitPrice` — 市价下单时，若无法取得行情（price == 0）会返回此提示

前端建议事项：
- 在提交前尽量在前端验证 `usdtNum`、`price`、`lever`、`tdMode` 的基本格式与范围，这会降低用户等待与回包错误。
- 在提交之前可在 UI 上向用户展示“估算手数（张）”，该估算使用与后端同样公式（但仅作预览），以避免用户对实际下单数量感到意外。

## 接口细节与示例
公开示例以 JSON body 形式给出。

1) 限价买入 — /v2/contract/buyLimit (POST)
- 描述：用指定 price 下限价买入（USDT 结算），后端用 usdtNum 与 price 计算张数并下单。
- 请求 body 示例：
```json
{
  "maincoin": "USDT",
  "tradcoin": "BTC",
  "usdtNum": "100",
  "price": "20000",
  "tdMode": "CROSS_OR_ISOLATED",
  "lever": "10",
  "profitPrice": "21000",
  "lossPrice": "19000"
}
```
- 成功示例：
```json
{ "code": 0, "data": { /* 后端返回的 data（controller 当前返回请求 params） */ }, "msg": "" }
```
- 常见失败：见上面的校验 key 列表。

2) 限价卖出 — /v2/contract/sellLimit (POST)
- 描述：同上，type 为 SELL。
- 请求 body 与 buyLimit 同结构。

3) 市价买入 — /v2/contract/buyMarket (POST)
- 描述：使用合约当前行情价（后端取 `contract.coin_last(maincoin, tradcoin)`）作为 price 进行计算与下单。
- 请求 body 示例：
```json
{
  "maincoin": "USDT",
  "tradcoin": "BTC",
  "usdtNum": "100",
  "tdMode": "CROSS_OR_ISOLATED",
  "lever": "10",
  "profitPrice": "21000",
  "lossPrice": "19000"
}
```
- 注意：若后端返回的行情价格为 "0"，接口会返回 `limitPrice` 错误。

4) 市价卖出 — /v2/contract/sellMarket (POST)
- 描述：同上，type 为 SELL。

## 建议给前端的 UX 细节
- 在前端做一次本地估算（采用相同计算公式：floor(usdtNum / (price * unit))）并展示给用户“预计下单张数”；提示最终数量以服务端计算为准。
- 在限价下单中展示价格与合约当前价的差异提示（后端对 BUY/SELL 有额外的上下界校验）。
- 若后端返回 `noEffective`，提示用户去激活合约交易权限或跳转到相关页面。
- 在下单之前，客户端可做小范围校验（非精确替代后端规则）以减少回包错误。

---

## 后端实现要点（供前端与后端对齐）
- 手数计算使用向下取整（RoundingMode.DOWN），按 `amountAccuracy` 保留小数位。
- 后端会做以下严格校验并返回相应的本地化错误文本：参数完整性、数字格式、交易对是否存在、unit 是否有效、min/max 检查、账户是否有合约交易权限和价格范围校验。
- 若需多语言展示，请以后端返回的 `msg` 为准（后端使用 `LanguagesUtil.get(key)` 返回文本）。

---

## 变更与注意事项
- 如果后端对 `tdMode`、`lever` 或枚举字符串有明确约定（如 `CONTRACT_MODEL_CROSS` 的实际字符串），前端请与后端确认并使用约定值。
- 推荐后端补充一个“预估”接口（例如 `/v2/contract/preview`），仅返回 computed `number` 与所需 USDT，供前端展示。当前没有单独预览接口，前端可自行用同样公式估算。
